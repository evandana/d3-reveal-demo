Promise Spec (1 of 2)

A Promise's states: unfulfilled, fulfilled, and failed

  • A promise represents the eventual value returned from the single completion of an operation.
  • A promise may be in one of the three states, unfulfilled, fulfilled, and failed.
  • Once a promise is fulfilled or failed, the promise's value MUST not be changed, just as a values in JavaScript, primitives and object identities, can not change.

http://wiki.commonjs.org/wiki/Promises/A

Promise Spec (2 of 2)

Implement a Promise's interface with these specs

  • This API only defines the necessary interface that a promise must provide for promise consumers to interact with it.
  • Implementors are free to define how promises are generated.
  • Some promises may be provide their own functions to fulfill the promise, and other promises may be fulfilled by mechanisms that are not visible to the promise consumer.

http://wiki.commonjs.org/wiki/Promises/A

Looks pretty similar, right?


                            // not runnable

                            function(object) {
                            // the object was saved.
                            },
                            function(error) {
                            // saving the object failed.
                            }

New Chain Pattern

Now using Promises

$.Deferred() Spec

    A constructor function that returns a chainable utility object with methods to
  • register multiple callbacks into callback queues, invoke callback queues,
  • and relay the success or failure state of any synchronous or asynchronous function.

$.Deferred() and Beyond

Libraries

  • Q Works in both NodeJS and browsers, compatible with jQuery, thenable, and usable as remote objects via message passing
  • RSVP.js A lightweight library that provides tools for organizing asynchronous code
  • when.js Compact promises with joining and chaining
  • node-promise Promises for NodeJS
  • jQuery 1.5 is said to be 'based on the CommonJS Promises/A design'.
  • ForbesLindesay/promises-a A bare bones implementation of Promises/A intended to pass https://github.com/domenic/promise-tests while being as small as possible
